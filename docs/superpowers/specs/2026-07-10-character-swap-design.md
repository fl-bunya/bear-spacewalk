# キャラクター差し替え機構 + 新キャラ「mspn」設計

日付: 2026-07-10

## 目的

現在カメラに同伴して漂う黄色い熊を、URLパラメータで別キャラクターに差し替えられるようにする。
元の熊はそのまま残し、既定の表示とする。第1弾の追加キャラとして `materials/IMG_9841.jpg` の
三面図（正面・側面・背面）を元にした「mspn」を実装する。

## 切り替え仕様

- `?char=mspn` で新キャラを表示。`?char=` 指定なし・未知の値は元の熊（既定）。
- UIは追加しない。既存の `?shirt=文字` と同じURLパラメータ方式に揃える。
- `?shirt=` は熊専用の機能であり、mspn では無効（シャツ自体がないため）。

## アーキテクチャ：キャラごとのビルド関数方式

現在 `index.html` にベタ書きされている熊の造形コード（約230行、部品生成〜輪郭線付与）を
`buildBear()` に関数化し、新キャラを `buildMspn()` として並置する。

```
buildBear()  ─┐
              ├→ { group, armL, armR, legL, legR, updateFace(t, forceBlink) }
buildMspn() ─┘          ↑ 共通インターフェース
?char= で選んだ1体だけ生成 → camera.add(group)
```

- アニメーションループ（腕バタバタ・脚キック・まばたき）は返り値のインターフェースだけを
  参照する。キャラによる分岐をループ内に持ち込まない。
- 共通部品は関数の外で共有する：`part()` ヘルパー、`gradientMap`、輪郭線
  （inverted hull）付与、宇宙飛行士装備（ヘルメット・首リング・酸素タンク）の生成。
- 生成するのは選択された1体のみ。両方生成して非表示にはしない。
- CONFIG の `bearOffset` / `bearXFrac` / `bearScale` / `bearColor` と、配置・照明
  （フィルライト）のロジックは両キャラ共通で流用する。

データ駆動（部品定義の配列）は2キャラには過剰（YAGNI）、if分岐の散在は可読性低下のため
不採用。

## mspn の造形（原画準拠）

- 体色: 熊と同じ黄色（`CONFIG.bearColor` = 0xf7d452）。服・シャツ・ロゴ帯なし、全身同色。
- 体型: 頭と胴の区別が薄いずんぐりの洋ナシ型。頭部球＋胴カプセルを近接・重ねて一体感を出す。
- 耳: 熊より小さめの球を頭頂に2つ。
- 腕・脚: 熊より短いカプセル。付け根を軸に回転できるようジオメトリをオフセット
  （熊と同じ手法）。
- しっぽ: 背面下部に小さな球。
- 輪郭線: 熊と同じ inverted hull 方式。
- 装備: ヘルメット（透明バブル）・首リング・酸素タンクを熊と同様に装着。

## mspn の顔

- Canvasテクスチャ（equirectangular、熊の `makeFaceTexture` と同方式）で固定描画。
- 内容: 下がり気味の短い線の目（左右）＋小さな口。鼻なし。
- まばたきアニメなし。`updateFace` は熊のみテクスチャを切り替え、mspn では何もしない。

## テスト・検証

- 目視確認: `?char=mspn` で新キャラ表示、指定なしで従来の熊表示、`?shirt=あ` が熊で
  従来どおり動くこと（リグレッション）。
- スマホ縦画面リサイズ時の配置ロジック（bearXFrac 由来）が両キャラで機能すること。
- 既存の screenshot.js / プレビューモード（`isPreview`）が mspn でも動くこと。
